# Source Preservation

## Principle

Do not rely on inline citations alone.

Preserve source text in separate markdown files so:

- the research trail is auditable
- later synthesis can quote or review the source again
- the user can inspect the original text without re-running search

## Source File Template

Each preserved source file should include:

```text
# Source 001

## Title

## URL

## Source Type
- article / report / paper / book excerpt / webpage / interview / policy text

## Accessed

## Why This Source Matters

## Original Text

## Notes
```

## Web Articles

Save:

- page title
- URL
- access date
- extracted text or the relevant excerpt

If the extracted page is very long, it is fine to preserve the relevant sections most tied to the research question.

## Books

Save:

- author
- title
- edition if known
- chapter or section
- relevant paragraph or passage

Do not try to reproduce large copyrighted sections that the user has not provided or lawfully accessed.

Prefer:

- the specific paragraph or passage relevant to the research point
- bibliographic metadata
- your note explaining relevance

## Source Index

Maintain `source-index.md` with:

- source id
- citation key
- title
- URL or bibliographic locator
- file path

## Distinguish Clearly

- `Original Text`: copied or extracted source material
- `Notes`: your interpretation or relevance summary

Never blur the two.
